# 为什么从"京东 AGI / 跨平台经营统一层"切到"专家对赌担保平台"

## 一句话
这个目录先后长出过三个本体，现在主线锁定第三个。前两个不是错，是路径，转前序归档。

## 三个本体的演化
1. 京东礼品店增长运营 AGI（最早）：把机会当成"单类目运营 know-how 工具"。问题：易被平台下沉吞噬，本质还是工具。
2. 跨平台经营统一层 / 企业采购 Agent（00-重定位、项目计划书）：把机会重定义为"多平台数据割裂"，做统一经营状态层。问题：偏数据工具，承重假设（割裂是真痛且单平台后台解决不了）尚未验证，且不是我们这轮收敛的方向。
3. 专家对赌担保平台（现主线）：从 OnlyFans（验真 + 直接变现）迁移 + Musk 第一性删一刀，落到"真专家验真 + 可控结果对赌 + 强制分账结算"。本体是结算信用，不是工具，也不是数据打通。

## 为什么是第三个
- 它解决的是陌生人之间的对赌信用，这层国内外没人同时做（验真+对赌+强制分账）。
- 切口最小最快可验真（京东礼品店投手 ROI 对赌）。
- 与第二个的根本区别：我们不打通一个商家的多平台数据，我们把同一服务结构一个平台一个平台复制。
