# AGENTS.md

## 仓库性质
本仓是 ZJPT（专家结果共担平台）独立执行仓。主线以 `ops/执行状态总表.md` 为当前闸门，以 `meta/ZJPT系统指令.md` 为项目运行源指令。

## Codex 进入仓库后的零号动作
1. 先读 `ops/执行状态总表.md`。
2. 再读 `meta/ZJPT系统指令.md`。
3. 涉及主线判断时，先确认是否已有滩头实验数据；没有数据不得把推演当结论。

## 文件路由
- `docs/`：对外主线文档。
- `滩头/`：承重墙实验。
- `过堂/`：SK / PB / LJKL 三引擎拆解。
- `研究指令/`：深度研究 prompt 与回收报告。
- `ops/`：执行看板与未验证教训。
- `meta/`：系统指令与仓库元信息。

## 操作纪律
- 不覆盖同名已有文件，先报告。
- 跨仓移动必须同时满足：ZJPT 写入内容、来源仓 `git rm`、来源仓留痕。
- 两个仓库分别提交，不混提交。
- 未经确认不 push。
- `meta/ZJPT系统指令.md` 是落盘源；如需修改系统指令，先贴改动让人确认。
