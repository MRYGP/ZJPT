# ZJPT · 专家结果共担平台

## 一句话
市场已定：独立站 / 跨境 DTC。当前模型是 agency 倾向的商家侧 operator：用可被验真的独立站投流真专家 + AI，替中国中小独立站卖家把 Meta / Google / TikTok 等海外平台投放做到位，先收服务费，可后置叠加小分成。

不是自助 AI 工具 / 不是撮合名单 / 不是大服务商全案。

结果共担转为后置战略分叉：独立站归因脏，当前不把结果共担作为地基；先验证 agency operator 是否有真实付费、达标、留存、可放大。

总闸前置条件：找到独立站投流真专家。专家到位前，项目待启动；未验证前不融资、不建平台、不扩张。

补充·专家的定义校准（360 类比）：
- 平台 AI 已替代"操作型专家"（比 AI 更会点按钮的人）；ZJPT 要放大的是"判断型专家"——知道该让 AI 做什么、何时别信 AI、这家店病根在哪的人。
- 类比：不懂底层参数，却能用 360 帮朋友把电脑优化得比他们自己点得更好——工具抹平执行，判断仍稀缺。ZJPT 的投流AI＝"投流界的 360"，卖的不是软件，是"会用 360 的那个人"被 AI 放大后能服务很多店。
- 故护栏"增强>替代"在此具体化：不替代判断型专家，用 AI 放大他，让一个好投手带得动很多店。
- 真威胁随之精确：京小通自称"会思考"，正把"判断"也往平台里收——ZJPT 的窗口是"判断溢价尚未被平台填平"的这段时间。

## 现状（2026-06-09）
方向层已切到独立站 / 跨境 DTC。当前唯一下一步是找独立站投流真专家（兼职优先）；专家到位前，不启动 concierge MVP。整套文档是指南针，不是地图。

## ⚠️ 承重墙未验声明
全套逻辑当前为推演（决策依据等级 X）。独立站 agency operator 语境下的承重问题只能由真实付费客户验证，桌面/深研验不了：
1. 付费意愿——独立站卖家是否愿为专家+AI operator 支付服务费。
2. 达标——服务后投放结果是否稳定达标，而非一次性波动。
3. 留存——一轮做完客户是否续约，去中介化是否挡得住。
4. 可放大——AI 轮子能否让一个真专家服务更多客户且质量不塌。
未跑出这些承重信号前，不融资、不建平台、不扩张。结果共担 / 小分成是后置战略分叉，不是当前总闸。
