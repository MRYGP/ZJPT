# ZJPT · 专家结果共担平台 · 技术报告

> v1.0 | 2026年6月
>
> **目标读者**：没有技术背景、正在主导或参与ZJPT项目的决策者和执行者。本报告不讲代码，只讲"平台要做什么事、靠什么工具做、做完后数据怎么走、有什么风险"。

## 第一章：ZJPT 是什么

### 1.1 一句话定义

> **ZJPT 是一个"经营结果契约撮合平台"**——让京东礼品类目的好店，找到经过验真的同类目投手，用7天投放ROI对赌契约验证双方是否值得继续合作，由平台托管资金和数据留痕，保护双方利益。

用一个更日常的比喻来说：

假设你家需要装修，你现在的困境是：你不知道哪个装修队真的好，装修队也不敢给你打包票，钱一交就变成单向信任博弈。

ZJPT做的事是：找到那些真正做过同类型装修、有案例有记录的工人，让他们先看你家，给出"我敢按完工质量收费"的承诺，把"定金"放进第三方账户，完工后按实际质量核算，再释放资金。

- 装修队 = 京东礼品类目投手
- 你家 = 京东礼品店
- "完工质量" = 7天付费投放ROI
- "第三方账户" = 支付宝担保交易

### 1.2 ZJPT 不是什么

| 不是 | 是 |
|---|---|
| 全店代运营平台（客服/选品/库存全包） | 只做付费投放ROI这一件事 |
| 知识付费/培训课程 | 真实结果对赌，不是学方法 |
| 泛撮合市场（谁都能接） | 严格验真，只接同类目真投手 |
| 平台承诺ROI结果 | 平台做规则和托管，不为结果背书 |
| 先开发技术系统 | 第一阶段全人工，信号成立再建系统 |

## 第二章：三个角色，各自做什么

ZJPT的运转涉及三个角色。理解他们各自的行动，是理解整个系统的前提。

| 角色 | 他是谁 | 他在平台里做什么 | 他得到什么 |
|---|---|---|---|
| 京东礼品店主 | 有真实GMV和库存的好店，但不确定投手是否靠谱，怕被坑 | 填写店铺入场信息，授权只投流/只查看子账号，预付成功分成上限进托管账户 | 7天可量化的投放ROI改善；如果未达标，预付款按规则退回 |
| 同类目投手 | 真实操盘过京东礼品/文创/定制类目的付费投放人，敢按结果收费 | 接受验真评分（最高10分），过7分后查看店铺诊断数据，决定是否进对赌档，执行7天投放 | 固定小服务费 + 达标后的分成；建立自己的平台信用档案 |
| ZJPT平台 | 规则制定者、验真执行方、资金托管协调方、争议处理方 | 做投手评分验真，对接店铺入场审核，生成对赌契约，协调支付宝担保账户，每日收集数据截图，期末结算分账，管理信用档案 | 从每单收取固定服务费；复购继续走平台，收取分成手续费；积累可信度数据 |

## 第三章：平台要完成的 7 件事

不管现在是人工处理还是未来有系统，ZJPT必须可靠地完成以下7件事。这7件事构成平台的最小闭环。

### 第1件事：店铺入场审核

**目的**：不接烂店，保护投手愿意继续对赌的意愿。

审核维度（6项，全部满足才入场）：

- 有真实运营的京东店铺（不是新注册空店）
- 近90天有真实GMV记录（有销售额，不是零蛋）
- 有足够的商品库存（投流来了能发货）
- 有客服能力（订单来了有人接）
- 有不少于X元的投放预算（具体金额第一单谈定后锁定）
- 愿意授权只投流/只查看子账号给投手，且愿意每日截图数据

> ⚠️ **注意**：不通过审核的店不进平台，不委婉，直接告知原因。拒绝烂店是平台最重要的风控动作之一。

### 第2件事：投手验真评分

**目的**：确保进入对赌档的投手是真正做过京东礼品类目投放的人，不是来碰瓷的。

评分卡（满分10分，7分以上才进"可对赌档"，4-6分进"固定费试跑档"，3分以下拒绝）：

| 评分项 | 分值 | 验证方式 |
|---|---|---|
| 有京东平台真实操盘经历 | 3分 | 操作截图 + 账号后台记录（脱敏） |
| 有礼品/文创/定制同类目案例 | 2分 | 店铺截图 + 数据报表（脱敏） |
| 能提供可验证的ROI数据（非P图） | 2分 | 后台截图与时间戳比对 |
| 有可联系的前合作店铺背书 | 2分 | 电话或微信验证 |
| 愿意接受7天按结果结算 | 1分 | 口头/书面确认 |

### 第3件事：冷诊断（看店不承诺）

在正式对赌之前，让投手查看3-5家候选店铺的基础数据，给出分类判断：

- **A类**：可对赌（投手愿意进7天ROI对赌档）
- **B类**：只适合固定费试跑（数据不够清晰，风险太高不适合对赌）
- **C类**：拒绝（店铺基础太差，投流也救不了）

拒绝的理由要记录。拒绝本身是平台信用的一部分：平台不会把烂店硬塞给投手。

### 第4件事：对赌契约生成与签署

每一笔对赌必须有一份书面契约，明确以下内容：

| 条款项 | 内容 |
|---|---|
| 基准ROI | 对赌前7天（或更长基准期）的实际投放ROI均值 |
| 目标ROI | 对赌期结束时需达到的ROI数值 |
| 对赌周期 | 7天（建议锚定在节日营销节点前，不做随机日历日） |
| 授权范围 | 投手能操作什么、不能操作什么（子账号权限清单） |
| 排除项 | 哪些情况导致对赌作废（如店主修改基础定价、平台大规模调权重） |
| 预付金额 | 店主预付的分成上限（放入支付宝担保账户） |
| 固定服务费 | 店主向平台支付的固定小额服务费（不退） |
| 达标释放 | 达标后分成如何分配（平台X%、投手Y%） |
| 未达标处理 | 未达标时预付款如何退回或部分退回 |
| 争议处理 | 争议发生后的证据收集流程和裁决规则 |

> ⚠️ **关键提醒**：第一阶段契约由人工起草，使用Word/飞书文档模板。双方微信确认即生效，无需律师公证。出问题时平台的争议处理依据这份契约，不是口头约定。

### 第5件事：数据留痕（每日收集）

对赌期间，需要每天收集以下数据，作为最终结算的依据：

- 京东商智/京麦后台：付费投放数据截图（花费、点击、转化、ROI）
- 投手操作记录：当日出价调整、计划开关、创意改动截图
- 店主确认：每日数据是否与截图一致（微信回复即可）

为什么要每天做？因为如果只在期末收集，双方很容易对数据产生争议（"你那天操作了什么"记不清了）。每日留痕是最低成本的争议预防。

> ✅ **第一阶段执行方式**：用飞书多维表格或腾讯文档表格，每天由投手填写，店主确认，平台存档。不需要任何代码。

### 第6件事：结算与分账

对赌期结束后，按以下流程结算：

1. 平台汇总7天数据，计算实际ROI与目标ROI的差距
2. 判断是否达标（全达标/部分达标/未达标）
3. 按契约规则生成结算单（截图+数字+结论）
4. 双方在结算单上确认（微信/飞书签字）
5. 平台协调支付宝担保账户释放对应金额
6. 平台收取服务费，投手收取分成，店主剩余预付款退回

### 第7件事：信用档案积累

每完成一笔对赌，平台为投手和店铺各生成一条信用记录：

- **投手档案**：参与对赌次数、达标次数、达标率、拒绝率、类目分布
- **店铺档案**：配合度（授权是否顺利、数据是否按时提交）、达标次数

这份档案是ZJPT长期价值的核心。投手的信用档案越厚，平台对他的议价能力越强；店铺的配合度记录越好，越容易吸引高分投手接单。

## 第四章：现阶段用什么工具（不开发系统）

ZJPT第一阶段的原则是：**用现有免费工具跑完整个闭环，不写一行代码，不花一分钱买技术服务。**

| 平台要做的事 | 工具 | 理由 |
|---|---|---|
| 店铺入场信息收集 | 腾讯问卷 / 飞书表单 | 免费、无需注册账号即可填写、数据自动汇总 |
| 投手信息收集与评分 | 飞书多维表格（创建10分评分卡） | 评分可直接填入，一目了然，支持多人协作 |
| 冷诊断沟通 | 微信 + 腾讯文档 | 最低摩擦，投手和店主都在用 |
| 对赌契约生成 | Word模板（事先做好，每次改数字） | 店主和投手都能打开，微信截图确认即生效 |
| 资金托管 | 支付宝担保交易 | 店主和投手都熟悉，出争议有记录，免费使用 |
| 每日数据留痕 | 飞书多维表格 / 腾讯文档共享表格 | 实时填写，双方都能看，自动留时间戳 |
| 结算单生成 | Word/飞书文档模板 | 每次填入实际数字，双方确认截图存档 |
| 信用档案管理 | 飞书多维表格（按投手/店铺分sheet） | 持续积累，不需要数据库 |
| 日常沟通协调 | 微信群（一组一群） | 每个对赌组建一个群，店主+投手+平台管理员 |
| AI辅助（可选） | 通义千问 / Claude / Kimi（均免费） | 用来验截图真伪、生成结算单初稿、汇总每日日报 |

> **为什么不先建平台系统？**
>
> 建系统要花2-6个月和几十万元。但ZJPT目前最大的未知数不是"技术能不能做"，而是"投手敢不敢赌、店主愿不愿预付"。如果人工跑出来都没有人愿意玩，再好的系统也是白费。先用人工跑出4个承重数，再决定建什么系统。

## 第五章：一笔对赌的完整数据流

下面是从"店主进入平台"到"一笔对赌结束"的完整数据流动路径。每一步会产生什么数据、由谁存档、存在哪里。

| 步骤 | 谁操作 | 产生什么数据 | 存在哪里 |
|---|---|---|---|
| 1. 店主填写入场表单 | 店主 | 店铺基本信息表（GMV、库存、预算、联系方式） | 飞书表单自动汇总 |
| 2. 平台审核店铺 | 平台管理员 | 审核结论（通过/拒绝+理由） | 飞书表格：店铺档案 |
| 3. 投手填写信息+接受评分 | 投手+平台 | 10分评分卡，附脱敏截图、背书联系方式 | 飞书表格：投手档案 |
| 4. 投手查看店铺做冷诊断 | 投手 | 诊断结论（A/B/C类 + 理由） | 腾讯文档：冷诊断报告 |
| 5. 生成对赌契约 | 平台管理员 | Word契约（含所有条款数字） | 微信群 + 本地存档 |
| 6. 双方确认契约 | 店主+投手 | 微信截图确认记录 | 微信聊天记录（双方各自存档） |
| 7. 店主预付进支付宝担保 | 店主 | 支付宝担保交易截图 | 飞书表格：资金状态 |
| 8. 投手每日操作+截图 | 投手 | 当日广告后台截图（花费/ROI）+ 操作记录 | 飞书多维表格：每日日报 |
| 9. 店主每日确认数据 | 店主 | 微信回复"确认"，截图存档 | 飞书表格：数据确认列 |
| 10. 期末汇总结算 | 平台管理员 | Word结算单（含7天数据汇总、达标判断、分账方案） | 微信群发送，飞书存档 |
| 11. 双方确认结算单 | 店主+投手 | 微信截图确认 | 平台存档 |
| 12. 支付宝担保释放资金 | 平台协调 | 支付宝转账记录 | 支付宝账单 + 飞书记录 |
| 13. 更新信用档案 | 平台管理员 | 投手/店铺信用记录+1条 | 飞书表格：信用档案 |

> **关键原则：数据留痕的目的不是监控，而是争议预防**
>
> 85%的争议源于"我不记得那天数据是什么了"。每日截图+双方确认这套动作，把这个风险降到接近零。第一阶段手动做，第二阶段可以用工具自动抓取数据。

## 第六章：AI 能做什么，不能做什么

ZJPT会用到AI工具（通义千问、Claude、Kimi等），但需要清楚边界。

**AI 可以做：**

- 核验截图基础真伪：比对截图时间戳、格式、水印是否一致（辅助，不能替代人工判断）
- 生成对赌契约初稿：把关键数字填入模板，减少手工录入错误
- 生成每日日报摘要：把投手填的原始数据整理成一段话发给店主
- 汇总7天数据、计算是否达标：减少算错的风险
- 生成结算单初稿：按契约规则自动填数字，人工复核后确认
- 分析投手验真材料：给评分卡的各项提供辅助判断意见（最终分数由人决定）

**AI 不能做：**

- 承诺ROI结果：AI不能对店主说"用了平台ROI能涨X%"，这是虚假承诺
- 替代投手做投放决策：AI不理解具体店铺、具体人群、具体节日节点
- 判断争议谁对谁错：争议判断需要人类理解上下文和商业常识
- 替代投手验真：真实性核验最后一关必须是人，不能完全外包给AI
- 管理资金：AI不接触任何资金流转，支付宝担保由人操作

> **一个判断原则**：AI负责"生成初稿+减少重复劳动"，人负责"最终判断+签字放行"。凡是涉及钱、合同、争议裁决的动作，AI只做辅助，不做决定。

## 第七章：四个承重数（最重要的部分）

在投入更多资源之前，ZJPT必须跑出以下四个数据。这四个数据是整个项目的信号灯。

| 承重数 | 定义 | 如何测量 | 解读方式 |
|---|---|---|---|
| ① 敢赌率 | 投手看完店铺诊断数据后，愿意进入"可对赌档"的比例 | 冷诊断结束后记录每个投手的分类结论，计算A类比例 | 低于30%：说明投手对店铺质量或规则设计不信任；高于50%：信号正常 |
| ② 赌赢率 | 进入对赌的店，7天内实际达标的比例 | 期末汇总结算单，统计达标/未达标 | 低于40%：目标ROI设定太激进，或店铺基础太差；高于60%：信号正常 |
| ③ 预付意愿 | 通过入场审核的店主，实际愿意把钱打进支付宝担保账户的比例 | 统计"审核通过"到"钱到账"的转化率 | 低于50%：店主信任度不足，或预付金额门槛太高；高于70%：信号正常 |
| ④ 复购走不走平台 | 第一单达标后，第二次合作是继续走平台还是绕开直连投手 | 第一单结束30天内电话/微信回访店主和投手 | 绕开比例高：平台在续约环节没有制造新价值，需重新设计复购留存机制 |

> ❌ **这四个数跑出来之前，不融资、不建系统、不扩品类。** 这是硬性原则，不是"条件成熟再说"。

## 第八章：主要风险和对应处理

| 风险 | 可能性 | 影响 | 对应处理 |
|---|---|---|---|
| 店主预付意愿极低（口头感兴趣，实际不付） | 高 | 整个对赌模式无法启动 | 降低首单预付门槛，试跑500-1000元；或先做固定费诊断服务积累信任 |
| 7天周期信噪比低（节日前后ROI差异极大） | 高 | 赌赢/赌输都不能说明投手能力 | 将对赌周期锚定在具体营销节点前（如节前7天），不做随机日历日 |
| 京东账号授权粒度不够（子账号权限太粗） | 中 | 投手无法精确操作，或数据无法精确归因 | 第一单前确认具体能授权哪些权限，写入契约排除项 |
| 投手在结算后绕开平台直连店主 | 中 | 平台无法建立复购价值 | 通过信用档案和"只有平台能持续提供新陌生店"来制造粘性；绕开率高则转型供给引擎 |
| 争议双方各执一词，平台无法裁决 | 中 | 损害任一方信任，平台口碑受损 | 每日数据留痕+双方每日确认；契约写清争议处理规则，平台做"证据整理员"，不做"法官" |
| 投手验真流程被绕过（伪造截图） | 低 | 平台信用崩塌 | "有可联系的前合作方背书"是最难伪造的一项，优先权重这项，电话核实 |
| 礼品类目本身季节性太强，非节日期找不到愿赌的投手 | 中 | 供给季节性断档 | 建立投手档案后，主动在节日前2-3周发起对赌邀约，不等待被动匹配 |

## 第九章：14 天启动路线图

以下是基于"先跑信号，再建系统"原则的14天启动计划。每天有明确的行动和产出。

| 天数 | 行动 | 预期产出 |
|---|---|---|
| 第1天 | 打3-5个电话给你认识的京东投手（哪怕是朋友的朋友），问：你敢不敢对陌生礼品店按ROI对赌？ | 原始反应3-5条；了解核心障碍是什么 |
| 第2天 | 根据电话反应起草：投手10分评分卡 + 店铺入场6项审核清单 | 两份文档初稿（Word） |
| 第3天 | 确定支付宝担保交易作为托管工具，测试一笔小额担保流程（自己给自己转1元测试） | 担保交易流程走通，知道操作步骤 |
| 第4天 | 起草对赌契约模板（Word），重点写清楚：基准、目标、预付、排除项、争议处理 | 契约模板初稿 |
| 第5-7天 | 访谈10个投手（不一定是陌生人）：评分卡测试，问5个核心问题，收集"愿意进可对赌档"的信号 | 10份投手访谈记录；至少2-3个"看店后可对赌"的候选人 |
| 第8-10天 | 同步访谈5家京东礼品店（弱关系优先，不强求陌生人）：入场审核+意愿摸底，重点问：你愿意预付多少？ | 5份店铺访谈记录；真实预付意愿数据 |
| 第11-12天 | 让1个通过评分的投手，对2-3家通过审核的店做冷诊断；记录诊断结论（A/B/C类） | 冷诊断报告；敢赌率初步数据 |
| 第13-14天 | 争取第1笔真实契约（弱关系，不强求陌生人）：固定服务费+预付+7天执行 | 第1单契约签署；第一笔预付款进托管账户 |

**14天后只做一个判断：**

- **有真实投手敢赌 + 真实店主预付 + 数据能授权** → 继续跑信号，争取第2、3单
- **投手不敢赌 或 店主不预付** → 转型固定费诊断服务，重新验证基础假设
- **首单后必然绕开且供给跟不上** → 放弃平台模式，转顾问撮合制

## 第十章：信号成立后，第一版系统长什么样

如果14天后四个承重数至少有3个信号正常，可以考虑建第一版轻量系统（不是完整平台，是"契约工作台"）。

| 功能模块 | 解决什么问题 | 技术复杂度 |
|---|---|---|
| 店铺入场表单（在线版） | 替代现在的飞书表单，数据自动进数据库 | **低**：用腾讯问卷Pro或Airtable即可，不需要开发 |
| 投手验真档案（结构化） | 替代现在的飞书表格，支持检索、评分历史 | **低**：Airtable或飞书多维表格即可 |
| 对赌契约生成器 | 输入关键数字，自动生成契约PDF，双方电子签名 | **中**：可用简道云/魔方网表实现，无需开发 |
| 每日数据自动抓取 | 替代投手手动截图，自动从京东商智API拉取ROI数据 | **高**：需要接京东开放平台API，建议第三阶段再做 |
| 结算计算器 | 输入7天数据，自动判断达标与否，生成分账方案 | **低**：Excel公式即可，后期可做成网页表单 |
| 信用档案（积累型） | 每笔对赌结束后自动追加一条记录，可导出报告 | **低**：飞书多维表格即可，数据量大后再迁移 |
| 争议证据包生成 | 一键打包该笔对赌的所有截图、日报、契约，生成PDF | **中**：可用Python脚本实现，无需大系统 |

优先级顺序：**结算计算器 > 信用档案 > 契约生成器 > 投手档案 > 店铺表单**。从最影响信任的环节开始做，不要从最酷的功能开始。

> **费用参考（第一版系统）**
>
> 如果完全使用无代码工具（飞书企业版 + 简道云 + Airtable），月费约500-2000元，无需工程师。如果需要数据自动抓取（京东API），需要外包开发，预估费用2-5万元，3-4周完成。在信号没有稳定跑出之前，建议选前者。

## 总结：三个最重要的判断

| 判断 | 内容 | 时间节点 |
|---|---|---|
| 第一判断：契约能否成立 | 陌生（或弱关系）投手和店主，是否愿意完成一笔有真实预付的7天ROI对赌。这是ZJPT存在的最基础前提。 | 第14天 |
| 第二判断：供给能否持续 | 平台能否持续制造新的"陌生人对赌对"，而不是帮熟人介绍完就没了。这决定了是平台还是撮合中介。 | 第3单后 |
| 第三判断：复购是否留在平台 | 首单达标后，第二单是继续走平台还是绕开直连。这决定了平台的商业模式是单次撮合还是信用平台。 | 首单结束后30天 |

> **最后一句话**
>
> ZJPT第一阶段不是在建平台，而是在证明一件事：陌生人之间，一个经过验真的投手，敢对一家好店的可控ROI打钱赌。这件事如果能在人工条件下成立，后面的每一步才有地基。
