# 朋友店 · 月一作战图（投流ROI改善 + 归因切法）

## 一、本图只回答一个问题（地基，先别问别的）
> **这家京东礼品店投流 ROI 改善了吗？改善能不能干净归因到 你/AI 做的动作（而非店本身流量好 / 平台红利 / 季节）？**

归因塌 → 验真专家 和 成功才收费 一起塌，后面全是空中楼阁。这比"小学模型的账"更早、更生死。

## 二、本图明确不碰（守 5-19 永动机防护）
分红结算 / 复购走不走平台 / 小学模型的账 / 单位经济 / 基础服务费定价 / 签约作家 / 平台 / 跨店可迁移性。
全部挂起，等月一跑出第一个干净 ROI 样本后再解锁。本图只做单店 0→1 地基。

## 四、归因设计（本图的核心，三层从强到弱）

### 主：同店内并行 A/B（净掉季节与平台红利）
- 同一账户、同一时间窗，分两臂并行跑：
  - **处理臂**：专家+AI 驾驭平台自带AI（京准通/京小通/全托管等）的计划簇。
  - **对照臂**：店主自己使用平台默认AI/全托管/一键托管的计划簇。
- 分臂维度：按 **SKU 组**（把可比商品随机分到两臂）或 **计划簇 + 人群包** 切开，预算上限对等。
- 为什么这是主：两臂同店同期、同一平台引擎，季节波动 / 平台红利 / 大盘流量对两臂作用一样，相减即抵消；差异只剩"谁在驾驶平台AI"——这正好切向驾驶溢价。

### 辅：前后同比基线（三角验证）
- 店自身历史 ROI 作前基线，并用 **去年同期同比** 折掉季节（礼品类目节日尖峰极强）。
- 单独用很弱（切不掉平台红利），只作主结论的旁证。

### 兜底说明：为什么不拿 on-off 脉冲当主
- 开关交替伤账户权重 + 时间混淆（学习期、节点）→ 仅在 A/B 无法分臂时降级备用，不作主证。

## 五、操作纪律（防数据污染）
- 专家用 **子账号** 按店主预算投，执行进平台可观测链路（解"执行不在供方手里"）。
- **专家不得碰对照臂**（否则偷偷优化对照臂 = 自己污染对照，结论作废）。
- 两臂预算对等、起停同步。
- **学习期处理**：30 天窗口，前 14 天算冷启动/学习（噪声段），用 **后 16 天** 评估。
  - 备选：若要更快出信号，可先跑 7 天快车（滩头设计提过），但须标注"未过学习期、仅作早期读数、不作判定"。

## 六、月一出口判定
1. **算什么**：后 16 天处理臂 ROI vs 对照臂 ROI。看 ① 改善幅度，② 是否稳定（多日均值，**非单日尖峰**）。
2. **改善阈值**：暂为占位，需杨拍板 或 首周用对照臂实测校准。建议形式："处理臂 ROI 稳定高出对照臂 ≥X%，且非单日尖峰"。（X 不凭空填——数据>推演。）
   - 判定如何翻译成"有没有意义"，见 `滩头/意义判据-月一口径对齐.md`。
3. **归因干净度自评**：逐条标【已控/未控】——
   - SKU 组本身可比吗（不同品自带不同转化率）？
   - 同账户两臂内部抢量 / 互相竞价推高 CPC 了吗？
   - 专家越界碰对照臂了吗？
   - 节点/活动（618 等）是否落在窗口内、对两臂是否对等？

## 七、本图不假装解决的坑（诚实留空，下田专门找）
1. 单店 → 跨陌生同类目店的可迁移性（下一阶段，不在本图）。
2. 改善阈值的绝对校准（首店只能给相对结论）。
3. 同账户内部抢量对 A/B 的污染到底多大（可能要被迫退回 on-off 或换跨账户对照）。
4. 礼品类目季节性是否强到连同期 A/B 都被节点扭曲。

## 八、跑完之后（解锁条件）
月一拿到第一个**干净的 ROI 样本** = 地基样本到手。
→ 此时（且仅此时）解锁谈下一层；并以这第一个真实样本为据，做 06-09 投流AI引擎分支的 AB，决定是否解冻判断层、回写 README 承重墙与 ops 看板。
