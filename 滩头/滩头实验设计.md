# 滩头实验设计

## 为什么是它
深研做得再多，进档率/达标率/预付意愿/复购走不走平台都是 X 级。这四个数只能下田验。本实验＝整个项目的总闸。

## 定位先钉死
加速器，不是医院。只接好店（入驻门槛：近 3 月 GMV > 类目均值），不救烂店——归因才干净、专家才敢进档。

## 配置
一个类目（京东礼品/文创/定制）、一个真投手、3-5 家陌生且达门槛的同类目店。
店＝L2成长期（月销5-50万）同类目好店；专家＝L2单环节投手（非全品类大神）；结算锚单环节快车ROI。
最最小可先做"7 天快车账户 ROI 结果共担"（比 30 天更快出信号、纠纷更少）。
托管先人工：腾讯文档 + 第三方代管 + 店主授权的权限受控子账户。不建平台、不做工具锁。

## 要测的四个数
1. 进档率：投手诊断后敢进共担档的店占比（验"同类目增长可迁移到敢进档"）。
2. 达标率：进结果共担的店周期内达标占比（投放侧可控口径：付费 ROI/CTR/入池，非全店 GMV）。
3. 预付意愿：店主愿不愿事前把"预估分红上限"预付入托管（二层结构成不成立的命门）。
4. 复购走不走平台：首单达标的店，第二单是回平台再结果共担，还是绕开平台直连投手。复购支点在专家侧：测的是专家愿不愿把复购钉在平台（因分红大、赖账险实、需平台强制分账与分账裁判），非测能否拦住店主。非kill-gate，是战略分叉探针。

第一块最干净试验田（有 know-how、真实意愿、现成"兼职运营"价格对照）。

## Go / Kill
- Go：四个数都过阈值（阈值在前 3 家校准后写回 ops 看板）；其中复购数有两条合格路径——① 复购走平台比例够高（结算护城河成立），或 ② 虽不走平台、但新陌生供给速度够快（供给引擎模型成立）。
- Kill：① 投手普遍不敢进档或店主普遍不肯预付 → "增长可迁移 / 二层结构"假设不成立；② 或复购普遍直连且新供给速度跟不上 → "价值可捕获"不成立。任一触发，退回固定费交付或换环节/类目。
