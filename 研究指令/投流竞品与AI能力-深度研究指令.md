# 深度研究指令 · 投流竞品全景 + AI 能力边界

## 〇、给研究执行者的话
你是电商投放（付费流量/广告投放，俗称"投流"）领域的产业分析师。目标：给出一份**可决策**的竞品与 AI 能力全景，服务于一个"自己下场做投流 AI、先服务单家京东礼品店"的项目。

硬规矩（违反则本报告作废）：
1. **中国优先**：默认主语是中国电商市场（京东/淘天/拼多多/抖音/快手/小红书），国外作对照与"轮子来源"。
2. **时效**：以 2025–2026 现状为准；旧信息须标注年份。产品名/功能/价格会变，凡涉及当前状态一律联网核实，不靠记忆。
3. **真实数据 > 推演**：规模、份额、价格、ROI 改善幅度等用可核实来源；查不到就写"未找到"，**不要用推演填空白**。
4. **区分事实与判断**：每个结论标【事实·来源】或【推断】。
5. **必列失败/退场案例**：哪些"AI 投流 / 智能投放"产品或服务商做死了、被平台自带功能吃掉了、被砍了，比成功案例更有价值。
6. **不要抄营销话术**：厂商自称"全自动 AI 投放"时，追问它真实能力边界与用户实测口碑。

---

## 一、研究问题（三大块）

### Block A · 投流平台本体 + 平台自带的 AI/自动化
列出主流投放平台，重点是**各平台自己内置的智能投放产品**（这是"AI 投流"的既有对手与天花板）。

- **国内**（锚点，需核实 2025–2026 现状与改名）：
  - 京东：京准通体系 / 快车 / 海投 / 京东直投等——**重点**，因为朋友店在京东。
  - 淘天：万相台无界、直通车、引力魔方。
  - 拼多多：多多搜索、多多场景、全站推广。
  - 抖音电商：巨量千川（含智能托管/全域推广）。
  - 快手：磁力金牛。
  - 小红书：聚光 / 乘风。
  - 微信视频号 / 腾讯广告：ADQ、视频号带货投放。
- **国外**（对照 + 轮子来源）：Google Ads（Performance Max / PMax）、Meta（Advantage+ Shopping）、Amazon Ads（Sponsored Products/Brands、AMC）、TikTok Ads、Microsoft Ads、Criteo、Shopify Audiences/Marketing、Pinterest、Snapchat。

每个平台要答：① 自带智能投放叫什么、AI 自动化到哪一步（自动出价/自动建计划/自动素材/全托管）；② 是否开放 API / 第三方能在多大程度上"接管"投放；③ 平台是否在主动收口（把第三方优化层吃进自带功能）。

### Block B · 服务店家的第三方竞品（国内外）
区分四类，分别给代表玩家、商业模式、收费方式、现状（活/缩/死）：
1. **代运营 / TP（人力为主）**：宝尊、丽人丽妆、若羽臣等——已知是退场/亏损样本，核实最新财报与去中介化趋势。
2. **投放优化 SaaS / 工具（国内）**：光云科技（超级店长/快麦）、店小秘、聚水潭、有赞、微盟，及淘宝/京东服务市场里的"智能投放/自动出价"工具。
3. **投放自动化 / AI SaaS（国外，轮子来源）**：Smartly.io、Skai（前 Kenshoo）、Madgicx、Revealbot、Optmyzr（Google）、Perpetua / Pacvue / Teikametrics（Amazon）、AdCreative.ai / Pencil（AI 素材）、Albert.ai，以及 2024–2026 新一波"广告 AI agent"。
4. **新型 AI agent / 自主投放**：能自主开计划、调价、复盘的 agent 类产品，国内外各列代表与真实成熟度。

每类要答：收费模式（佣金/订阅/分成/效果付费）、是否做过"效果付费/ROI 分成"、做这件事的死法。

### Block C · AI 目前能做什么、解决哪些问题（按能力维度）
逐维度给"已成熟 / 半成熟 / 还做不到"，并附真实证据或实测口碑：
1. 素材生成（投放图/视频/文案 AIGC）
2. 出价与预算分配（自动竞价、跨计划调预算）
3. 人群定向 / 选品 / 关键词拓展
4. 计划搭建与全流程托管（开计划→投放→调优→复盘）
5. 数据分析 / 诊断 / 报告自动化
6. 跨平台统一投放与归因
7. **明确列"AI 当前做不到/做不好"的**：干净归因、冷启动、平台 API 限制、需要人介入的环节。

## 二、必须回答的 ZJPT 回写问题（把研究钉到我们的决策）
1. **差异化空间**：既有"AI 投流"（尤其平台自带 PMax / 万相台无界 / 巨量千川）已做到哪一步？一个新做"投流 AI"的项目，第一步差异化空间还在哪？还是已被吃光？
2. **可整合的成熟轮子**：哪些第三方工具/API 是"拿来即用的轮子"（演进路径第二步：整合别人成熟轮子），各自能省什么、坑在哪？
3. **巨头引力威胁（6-04）**：平台自带自动化是否已是 ZJPT 投流地基的最大威胁？有没有"巨头自带功能把第三方优化层吃掉"的真实尸体？
4. **归因方法借鉴**：业界给单店/单账户投放做 ROI 归因，用什么方法（A/B、holdout、增量测量 incrementality、PSA 对照等）？→ 直接喂养朋友店月一作战图的归因设计。
5. **效果付费/ROI 分成的现状**：有没有人在投放领域做"按效果/ROI 分成"并活下来？怎么活的？（呼应风险代理律师那条 capped survivor）

## 三、输出格式
- 三大块各一张对照表（玩家 | 模式 | 收费 | AI 能力到哪步 | 现状[活/缩/死] | 来源年份）。
- 单设一节「失败/退场案例」。
- 单设一节「对 ZJPT 的五条回写答案」（直接回答第二部分）。
- 全文每结论标【事实·来源】/【推断】；末尾附来源清单（含访问年份）。

## 四、不要做的事
- 不要把厂商自述当能力事实。
- 不要漏掉"平台自带功能吃掉第三方"的趋势。
- 不要用推演填补查不到的数据——直接标"未找到"。
- 不要替我们下结论说"该做/不该做"——只供事实与判断素材，决策留给项目。
