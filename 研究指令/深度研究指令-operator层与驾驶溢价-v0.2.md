# 深度研究指令 v0.2 · 商家侧 operator 层 + 驾驶溢价 + 平台填平威胁

## 〇、给研究执行者的话（先读，定位别跑偏）
被研究的项目**不造投放算法引擎**。它做的是"**商家侧 operator**"：用专家+AI，替商家把电商平台（京东等）**已经很强的**自带投放 AI（京准通/京小通/全托管/千川全域等）**用到位、用出超过店主自己默认用法的效果**。
- 平台 AI 是它要驾驭的发动机，**不是竞品**，不要去研究"谁的算法更强"。
- 它真正要回答的是三件事：① 同样在帮商家用平台 AI 的玩家有谁、活得怎样；② "熟练驾驭 vs 默认一键"到底有没有效果差（驾驶溢价）；③ 平台把操作一键化、自己下场做 operator（如京小通）的进度有多快。

硬规矩（违反作废）：中国优先（主语＝京东等中国电商）；2025–2026 时效、注明来源年份；真实数据>推演，查不到写"未找到"；每结论标【事实·来源】/【推断】；**必列死亡/退场/被替代案例**；不抄厂商话术，追问真实能力边界与店主实测口碑。

---

## 一、研究问题（三大块，全部围绕 operator 层）

### Block A · 商家侧 operator 层有谁、什么模式、活得怎样
把"帮商家做投放/经营"的玩家按类型扫一遍（中国优先）：
1. **代运营 / TP（人力 operator）**：宝尊、丽人丽妆、若羽臣、壹网壹创、青木、凯淳，及未上市的代投工作室/MCN 代投。要答：收费模式（服务费/扣点/分成）、2025 最新盈亏、为什么普遍承压（流量碎片化？CAC？买量利润被挤？）、活下来的（如若羽臣）靠什么活。
2. **平台自家 operator 助手（最关键）**：京东京小通、阿里 AI 小万、千川智能托管等。要答：它们已经能替商家做到哪一步（开计划/调优/诊断/一句话托管）？**对"店主自己就能开好"的门槛降到多低了？** 这是判断 operator 层会不会被平台填平的核心。
3. **第三方投放 SaaS / 工具型 operator（国内）**：光云科技（超级店长/快麦）、店小秘、聚水潭、有赞、微盟，及京东/淘宝服务市场里的智能投放工具。要答：是真"帮你投得更好"还是只做表层自动化。
4. **海外同构参照**：跨平台 AI agent operator（Synter、Albert.ai、Percuity、Madgicx 等）"你定方向、agent 执行"的模式、收费、成熟度——作模式参照（注：它们绑西方平台，接不进京东）。

### Block B · 驾驶溢价：熟练驾驭 vs 默认一键，到底差多少（命门外部旁证）
这是本项目的生死命门，研究要找**外部证据**：
- 有没有公开数据/案例/实验，对比"同一平台上，熟练运营/专业代投 的 ROI vs 默认全托管/小白自投 的 ROI"？差距有多大、在什么品类成立、什么品类不成立？
- 海外那项"640 次增量测试显示平台全托管长期跑不赢人工"的研究，原始出处、方法、结论边界是什么？可信度如何？
- 反面证据：有没有"平台全托管已经好到人工没法再优化"的证据？（若有，是对本项目的重大警告）
- 在京东/礼品类目上，有没有任何关于"投得好 vs 投得差"差距的具体数据？

### Block C · 平台填平威胁 + operator 价值缝隙
1. **填平进度（真威胁）**：京小通/AI 小万这类平台助手，过去一年把哪些原本要人做的 operator 工作收编了？店主自助化趋势的方向与速度？有没有"平台助手上线后第三方代投/服务商业务被挤掉"的案例？
2. **价值缝隙（活路）**：平台 AI 明确做不到、必须人+工具补的环节有哪些——干净归因/增量判断、跨场景经营决策、上手门槛与信任、素材策略、大促节奏？给证据。
3. **operator 层的死亡案例**：帮商家投放/代运营的玩家是怎么死的或被替代的（区别于"投放算法产品"的死法）。

## 二、必须回答的 ZJPT 回写问题
1. **operator 层是不是一个真实存在、且还没被平台填平的商业位置？** 证据是什么？
2. **驾驶溢价存在吗、在京东礼品这类场景大概多大？** 外部能找到的最接近的数据是什么？
3. **平台把 operator 填平的速度**：京小通等让"店主自己开好"的门槛，是在快速下降还是仍有显著鸿沟？给时间线判断。
4. **专家+AI 这个 operator 形态，比纯人力代投、比平台自助，分别强在哪、弱在哪？**
5. **归因旁证**：业界给"同平台、同店、不同操盘者"做效果对比时，用什么方法绕开样本量不足？（喂月一作战图）

## 三、输出格式
- Block A 一张 operator 玩家表（类型｜代表｜模式｜收费｜2025 现状［活/缩/死/被替代］｜来源年份）。
- Block B 一节"驾驶溢价证据"（正反证据分列，标可信度）。
- Block C 一节"平台填平进度 + 价值缝隙"。
- 单设"死亡/被替代案例"节。
- 单设"对 ZJPT 五条回写"节。
- 全文标【事实·来源】/【推断】，末尾附来源清单（含年份）。

## 四、不要做的事
- 不要回去研究"谁的投放算法更强"（定位无关，已废）。
- 不要把平台/厂商自述当能力事实，尤其"驾驶溢价"和"填平进度"要交叉验证。
- 查不到的数据标"未找到"，不要推演填空。
- 不替项目下"该做/不该做"结论，只供事实与判断素材。
