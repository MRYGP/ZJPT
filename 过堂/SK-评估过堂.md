# SK 评估过堂 · 专家对赌担保平台

## 0. 那一刀（评估结论的一句话）
它最值钱的动作——让陌生好店和锁死在外地的真专家完成"第一次敢赌"——同时也是它唯一能干净收费的时刻。第一单成功后双方不再是陌生人，关系顺着信任流出平台。支付宝类比不成立：支付宝服务的是"一次性陌生交易的长尾"，代运营是"高价值、强关系、重复博弈"，恰是支付宝模型最不适用的地方。

→ 本体不是"结算基础设施"（锁不住老关系），是"持续制造可信新陌生对"的供给引擎。天花板由**新供给速度**决定，不由"老关系锁多久"决定。这不是否决，是把它从"墙圈式平台"重新归类为"供给引擎式平台"——后者也能活（猪八戒/Upwork 是这一类），但要按这一类来测、来防。

## 1. 四要素判据打分

| 要素 | 评级 | 依据 |
|---|---|---|
| 垂直够小 | 🟢 强 | 切口=京东·礼品类·投手·ROI 对赌，单平台单类目单动作，反馈环天/周级。切入成本与对赌干净度重合，非巧合。|
| 人不可替代 | 🟡 中 | 真·同类目投手（骨科医生式）确实 AI 不可替代。**但承重检查不过：这个"人"不是 SK 的人，是平台招募的独立承包人。** 一旦首单建立信任，专家与店之间就不再需要平台——价值会从这里漏。|
| 动作可接管 | 🟡 偏 🔴 | **创造价值的动作（投放）被专家接管，不是被平台/AI 接管。** 平台自己接管的动作=验真+托管+分账+归因。其中托管/分账要么需支付牌照（SK 拿不到），要么是骑在第三方银行账户上的薄层（可复制）。"动作可接管"在平台这一侧站不实。|
| 价值可捕获 | 🔴 弱 | **两条承重机制双双有洞。**（1）"无动作可接管则价值可捕获为空"：平台的接管动作是牌照门或薄层。（2）强制分账只约束流经托管的钱——首单后双方互信，钱绕开托管。大头收入（溢价分红 20–30%）还压在两个 X 级假设上：归因可干净 + 店主肯预付。|

**两条承重机制专项结论**：四要素里 SK 标记为"承重"的两条（动作可接管→价值可捕获；人不可替代须是"你的人"），**两条都有结构性洞**。这不是"再优化文案能补"的洞，是模式本身要正面回答的：首单成功后凭什么留在平台。

## 2. 市场基底五问

| 问 | 判断 | 等级 |
|---|---|---|
| ① 数据类型 | 投放数据=持续变化型（日级 ROI 反馈），利好快速出信号。切口"最干净"正源于此。| B |
| ② 市场轨迹 | 代运营整体成熟偏收敛（品牌自营+平台降本），但"好小店缺好运营且不敢信"的长尾痛持续存在。轨迹混合，需数据。| C/X |
| ③ 付费方财务方向 | 目标=好店（GMV>类目均值）。**痛深与付费能力负相关在此显形**：好店有钱付，但也最不急、最容易直接挖人/直接合作。付费方=最能跳单方。| C |
| ④ 官方免费数字化风险 | 对"增强商家自用 SaaS"层风险高（微盟/有赞/阿里已占）。但对"陌生人跨店对赌结算"这一特定层，平台无动机自建——**此层风险较低，差异化成立**。相邻风险：京东服务市场已有"认证服务商+担保交易"雏形，可能向此延伸。| B/C |
| ⑤ 现有玩家锁定捕获 | 宝尊/丽人丽妆=大牌项目制；猪八戒/威客=单次任务红海；平台服务市场=认证撮合。"验真+对赌+强制分账"三件一起做确实空白。**但三个组件各自都存在，组合的防御=结算护城河，而护城河在复购环节泄漏。**| B/C |

## 3. 反 FM 审问 · TOP 3 风险

**FM003 伪需求陷阱（致死率 90%）— 第一承重墙**
- 机制：店主事前愿为结果付费、事后翻脸赖账；专家不敢赌。整套二层定价的命门=店主肯不肯**事前预付"预估分红上限"入托管**。今天是 X 级。
- signal_source：致命变量（预付意愿）＝滩头第 3 个数本身；代运营投诉平台搜"分红纠纷/赖账"看是否结构性集中。
- 现状：BP 已识别且把它路由进滩头，正确。

**FM008 竞品快速复制 / 反跳单（disintermediation）— 第二承重墙（BP 低估）**
- 机制：首单成功后专家+店关系流出平台；若托管层免牌照则是薄层可复制。BP 用"信用复利+支付宝从不禁止你付现金"判它"不再是威胁"——**此判断不成立**：支付宝适用一次性陌生长尾，不适用高价值重复关系。
- signal_source：边缘玩家（访谈≥3 个真实在操盘的投手，看他们与老客户首单后是否转直连）；"BP 是否在讲规模故事而非真实解法"自检。
- 现状：**滩头未测此项。这是本过堂要补的洞。**

**FM009 监管/资金合规（致死率 85%）— 落地前置闸**
- 机制：托管+强制分账+溢价分红收取，可能触支付牌照；自称担保=连带无限兜底；子账户越权=代运营无资质定性。BP 全部标【需律师确认】，方向对，但未闭环。
- signal_source：京东服务市场是否已提供"担保交易"（相邻吞噬信号）；律师就"第三方银行监管账户+里程碑分账、SK 不持牌"出具可行性。
- 现状：此条是红线①与"问题"之间的岔路。律师确认前不得扩张到朋友店之外。

（次级：FM006 平台依赖——子账户机制依赖京东/抖店官方政策，监控即可，非 TOP3。）

## 4. 要解决的问题清单（建造 > 否决；团队规模不是判据；增强 > 替代）

红线检查：不触发——**条件是**资金托管做成"第三方银行监管账户、SK 不碰本金不持牌"。若律师判定该结构仍需 SK 持牌 → 落入红线①，届时再否决；在此之前是要解决的问题。

1. **[承重墙一·X级]** 同类目增长可迁移 → 敢赌率/赌赢率/预付意愿。只有滩头能答。BP 已设计，正确。
2. **[承重墙二·本过堂新增]** 首单成功后凭什么留在平台？**滩头补第 4 个数：复购是否走平台（首单达标的店，第二单回平台 vs 直连投手）。** 若高比例直连 → 模式=供给引擎（按新供给速度建模、按"持续制造新陌生对"防御），不是结算护城河。两种都能活，但必须先知道是哪种。
3. **[资金合规·前置闸]** 律师确认"第三方银行监管账户+里程碑分账、SK 不持牌"是否成立。这是红线①与问题清单的岔路口。朋友店滩头可人工托管先跑，扩张前必须闭环。
4. **[担保定性]** 合同对外定性"托管分账工具"，不自称担保人，专家=独立承包人；律师确认连带责任边界。
5. **[代运营定性]** 子账户权限框死（仅投流/仅查看），核各平台权限粒度，避免越权落回代运营定性。
6. **[归因脏/ROI 非全控]** 只锚投放侧可控口径+承接基线门槛+拒赌权——已设计，滩头验证。
7. **[平台依赖 FM006]** 子账户机制依赖京东/抖店官方政策，监控政策变动。

## 5. Go / Kill（在 BP 基础上修订）
- BP 原 Go：敢赌率+赌赢率够高、店主肯付溢价。
- **本过堂追加 Go 条件**：复购走平台的比例够高（阈值滩头校准）；或虽不走平台、但新供给速度足以支撑供给引擎模型。
- Kill：① 投手普遍不敢赌或店主普遍不肯预付；② **或复购普遍直连且新供给速度跟不上**（价值捕获不成立）。
